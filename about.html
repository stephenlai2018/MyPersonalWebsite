< 中文自傳 >

    我31歲才出社會, 比一般人晚很多, 除了出國留學外, 其餘的時間都渾渾噩噩, 像一朵浮萍, 毫無目標的隨波逐流. 高中及大專讀社會組, 當時台灣科技電子業風靡國際, 光華商場是學生必去的場所. 我在那時迷上了電子, 雖然不懂原理, 但仍然省吃儉用的買了許許多多的電子零件包及工具回家焊接. 看到LED燈一閃一閃, 蜂鳴器發出尖銳的聲音, 讓我怦然心動, "原來這個世界有這麼神奇的東西叫做電子零件, 電路板, 電路圖", 從此種下了我出國學習的種子. 在美國堪薩斯州求學四年, 專攻電子工程, 我尤其對寫韌體控制微控制器非常感興趣, 一顆單晶片微控制器就能點亮LED, 控制馬達轉動, 讓蜂鳴器發出聲音, 還可以讀取各式各樣的感應元件 實在讓我欣喜若狂, 我大多數的時間都花在了解學習這件神奇的事和過程.

    回國後很順利地找到一家半導體公司擔任韌體工程師的工作, 不過並沒有做很久, 不到一年的時間就離職了. 然後很快地找到另一個軟體工程師的工作, 然而也待不久. 不到一年我又離開了. 之後同樣的情形繼續發生, 總共五份工作每個都維持不到一年. 到底發生甚麼事? 為什麼我都待不久? 其實現在回想起來, 就會有很深的感觸. 第一份工作其實我能勝任, 因為公司用8051單晶片製作開發板, 我在美國學習Atmel單晶片, 要切入不同微控制器並不會很困難. 而我也順利上手. 不過後來主管交付我一件任務, 要開發一個帶有作業系統的Demo板, 這對我來說是天方夜譚. 首先我並沒有學過計算機科學, 對作業系統毫無概念, 所以顯而易見的我根本無力完成這項任務, 於是在困惑中離職了. 第二份工作從事智慧型手機驅動程式軟體維護. 這對我來說也是一個新的領域, 不過有一兩位同事甚至完全沒有軟體背景也加入開發團隊, 讓我信心增長不少. 不過因為只知道皮毛, 專業知識技術不足, 看C++程式代碼像看外星語言, 所以毅然而然離開.

    從這幾個軟韌體開發工作經驗, 我歸納出以下幾點我做不長久的原因: 第一: 除了在學校學得的專業知識技術外, 進入業界接觸到的是更為複雜龐大的系統及各種不同的軟體及開發技術, 所以要不斷的學習, 這點我就沒有做得很好. 我除了從網路, 那時Google還不是主流, 我用Yahoo或Bing搜尋引擎搜尋到的資料都不能切中需要, 得到的很多都是錯誤的資料. 也因此無法突破進展. 第二: 我並沒有利用下班時間去上課, 所以新的技術比如作業系統, C++語言 只有一知半解, 完全狀況外. 第三: 我在國外因為用英文學習, 所以有許多的觀念並不真正了解, 一旦遇到問題就卡住了. 第四: 在國外學習我偏重知識上的, 我考試都很高分, 然而我實驗課卻不重視, 有一部分原因是不懂實驗目的何在, 這跟英文理解程度有關. 第五: 我不懂為何而戰, 為誰而戰; 不懂工作的意義是甚麼? 不懂許多人拼命努力工作賺錢養家背後的動機和目的.

    檢討完畢後, 接下來的六年我在鐵路局度過, 這段期間正值新人進入挑戰資深員工, 不論學歷高低, 年紀及資歷, 都產生一定程度的火花. 也算是一段難忘的體驗. 為了不讓資深員工看扁, 我硬是待了六年多的時間, 不過這裡畢竟不屬於我的世界, 該走還是要走的. 我休息三年多了, 這三年中我善用網路開源資料, 大量學習多樣技術, 包括一直讓我耿耿於懷的作業系統及C++程式語言, 及蓬勃發展的 Arduino, Raspberry等開源項目. 我深深了解網路科技的進步及重要性, 所以透過 Udemy 及 Youtube 等頻教學網站大量學習了網路技術如 HTML5, CSS3, Javascript 等前端技術, 我了解 Javascript 是瀏覽器(網頁)唯一程式語言, 其重要性不言可喻. 所以我就選擇走 Javascript 路線, 從前端到後端, 遊戲到手機, Javascript 已經變成最受歡迎, 使用人數最多, 進化最快, 支援最多的程式語言. 而且各式各樣的框架和程式庫, 在"Write Less, and Do more", 或"Write once, run everywhere" 的前提下讓網頁技術滲透到所有角落. 調查顯示 Github 專案下載次數最多的是由 Javascript 開發的, 高達10幾億多次, 由此顯示出 Javascript 語言的重要性.

    現在的我從零開始, 從心並且重新出發, 拋開一切煩惱憂愁, 忘記背後, 努力面前, 向著標竿直跑. 我希望能透過面試讓我被了解, 我也會盡最大努力工作, 不管是為公司獲利, 為個人經濟需求, 或為獲得成就感, 別人的肯定或自我價值的實現. 我的人生才開始, 未來是美好的.
